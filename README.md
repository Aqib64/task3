# WeatherApp — Task 3 (Material Design Redesign)

This is a COMPLETE project — same as your Task 2 app, but with the Material
Design redesign already applied, and your API key already filled in.

## Step 1 — Open it
1. Open Android Studio
2. Click **"Open"** (not "New Project")
3. Select this whole `WeatherApp_Task3` folder
4. Click OK, then wait for it to sync (progress bar at bottom)

## Step 2 — Run it
1. Click the green ▶️ Run button
2. Pick your emulator (Pixel 10 Pro XL, same as before)
3. Once it opens, search a city like `London`

You should see the new look:
- Outlined input box with floating label
- Blue Material-style button
- Light blue rounded box around the weather result
- Light red rounded box if there's an error

## Step 3 — Take your "after" screenshot
This is your Task 3 "after" — your Task 2 screenshot (the plain version)
is the "before".

## Step 4 — Submit
1. Push this project to GitHub (either as a new repo, e.g. `WeatherApp-task3`,
   or update your Task 2 repo with these changes — either is fine)
   - In Android Studio: **Git menu → GitHub → Share Project on GitHub**
   (if new repo) or **Git menu → Commit → Commit and Push** (if updating
   the same repo)
2. Submit deliverables:
   - Before/after screenshots
   - The `activity_main.xml` file counts as your design file
   - GitHub link showing the implemented redesigned screen
